Why it resists splitting · scope decisions

Parallelism caps around 2 on the critical path

The data layer (#2–#5) is a sequential gate — page work churns against mocks until it stabilizes.

The Explorer detail tabs (#10–#13) are one coupled surface — shared tab shell, selection store, and Thing container — so splitting tabs across devs adds merge contention faster than throughput.

Peel-off work — vendor boxes (#7), primitives (#14), catalog/docs — is where extra devs add value.

Scope locked 2026-06-29

Explorer: full parity — 9 tabs × 6 models, PDU grid + heatmap.

Writes: included — control actions via the voting workflow.

Widgets: full vendor parity + DCS.

API: match the reference app exactly — /list-things, /tail-log, /pdu-layout, /global/data, /actions/* — not the newer /auth/* aggregates.

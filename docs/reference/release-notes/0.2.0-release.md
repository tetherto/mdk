# Release v0.2.0

Released: June 2026

## Overview

MDK v0.2.0 is the first SemVer-aligned monorepo release after the architectural restructure. It delivers a federated 
three-domain layout (`backend/core`, `backend/workers`, `ui`), a formal worker protocol contract, Zustand-based UI state, 
React 19, and an agent-first CLI with MCP support on the app-node.

This release supersedes the pre-release `0.0.x` versioning used during early development. See [v0.0.1 release notes](./0.0.1-release.md) for the inaugural UI-focused release history.

## Highlights

First runnable end-to-end stack — backend wired to UI.

**BE:**
- ORK rewrite — DHT discovery, HRPC transport, IPC gateway, telemetry collector, command dispatcher, health monitor
- New MDK API — `getOrk`, `startWorker`, `startAppNode`, `waitForDiscovery`, `startServices`
- Worker contracts — every worker ships `mdk-contract.json` with telemetry, commands, health, and error catalogues
- Worker catalogue — miners (Antminer, Avalon, Whatsminer), containers (Antspace, Bitdeer, Microbt), minerpools (Ocean, F2Pool), power-meter (abb, satec, Schneider, electricity), temperature (Seneca)
- App-node MCP endpoint — AI agents can query fleet state and issue commands
- Services module — PM2 and Docker orchestration via `startServices(config)`
- Backend examples organized by device family (miners, containers, minerpools, power-meter, temperature)

**UI:**
- Package split — `@tetherto/mdk-ui-core`, `@tetherto/mdk-react-adapter`, `@tetherto/mdk-react-devkit`, `@tetherto/mdk-ui-cli`
- Zustand stores replace Redux Toolkit slices
- React 19 across all UI packages
- Agent-first CLI — `mdk-ui create`, `add page`, `suggest`, registry discovery, and IDE seeding via `mdk-ui init`
- Agent-ready build pipeline (`build:registry`, `check:agent-ready`, `registry.json`)
- Catalog app — `@tetherto/mdk-catalog-ui` replaces the legacy demo app

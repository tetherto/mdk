# Release v0.5.0

## Overview

v0.5.0 is a **naming, transport, and packaging** release. It completes the control-plane rename (**ORK → Kernel**, **App Node → Gateway**) across the whole stack, **retires the IPC transport** in favour of an HRPC key-file bootstrap, and finishes promoting the Worker runtime into a standalone **`@tetherto/mdk-worker`** package with every worker migrated onto the `WorkerRuntime` plugin model. On the UI side it renames `@tetherto/mdk-ui-core` to `@tetherto/mdk-ui-foundation` and restructures the devkit into `primitives`/`domain` layers. It also lands Gateway-side pagination/search, an Op Centre / Explorer UI data layer, a third-party Worker developer guide, a full-site dashboard + MCP example, and a broad test-coverage and mock-reliability push.

For the detailed, itemised list, see the [v0.5.0 changelog](../../../CHANGELOG.md).

## Breaking changes

- **ORK → Kernel**: `@tetherto/mdk-ork` → `@tetherto/mdk-kernel`, `backend/core/ork/` → `backend/core/kernel/`, `ORKManager` → `KernelManager`. The MDK protocol action names are unchanged, so 0.4.x peers still interoperate on the wire.
- **App Node → Gateway**: `@tetherto/mdk-app-node` → `@tetherto/mdk-gateway`, and the bootstrap export `startAppNode()` → `startGateway()`.
- **IPC transport removed**: HRPC is the only client transport. The Kernel now writes its HRPC public key to a key file (`<tmpdir>/mdk/.kernel-key`) so out-of-process clients connect config-free; `opts.keyFile` overrides, `keyFile: false` disables.
- **Worker runtime + packaging**: `WorkerRuntime` moved into `@tetherto/mdk-worker`; every worker migrated onto the plugin model and the legacy `base/`/`ThingManager` packages were removed. All worker packages renamed to `@tetherto/mdk-worker-*`.
- **UI**: `@tetherto/mdk-ui-core` → `@tetherto/mdk-ui-foundation`; devkit layers `core`/`foundation` → `primitives`/`domain`; the `styles-foundation.css` export renamed to `styles-domain.css`; the `./core`/`./foundation`/`./feature` subpath exports removed; registry/docs-data schemas bumped to `2.0.0` (`orkCapabilities` → `kernelCapabilities`).
- **Aggregation moved Kernel → Gateway**: the Kernel's `listThings`/`getThingsCount` responders were removed; the Gateway now aggregates cross-worker.

## What's new

- **Gateway pagination + search**: `/auth/pools` accepts `search`/`offset`/`limit`; containers and things listings gained server-side filter/sort/pagination backed by a new Gateway things aggregator.
- **`@tetherto/mdk-worker`**: standalone Worker Runtime (`WorkerRuntime`, `loadPlugin`, service builtins, a formal `mdk-contract` JSON Schema, opt-in zero-device boot).
- **Worker developer guides**: [build-a-worker](../../guides/workers/build-a-worker.md) and [build-a-minimal-dashboard](../../tutorials/quickstart/build-a-dashboard.md), plus an HRPC inspection guide and a runnable worker-caller demo.
- **Examples**: a full-site dashboard + MCP server, a new `site-backend` multi-process example (PM2/Docker), and a plugin-authoring e2e.
- **Tooling**: a `docs:generate` pipeline with versioned reference nav, a new `examples` CI pipeline, and a `.mailmap`.

## Removed

- The **microbt** container workers and the **electricity** power-meter worker.
- The worker **`base/`** packages, the Kernel **`DataAggregator`**, the client **IPC** transport, and the **`mdk-utils`** package (absorbed into `@tetherto/mdk`).

## Fixed

- Paged `listThings`/`cabinets` no longer hit the removed Kernel RPC; containers list no longer truncates or miscounts `total`.
- Deterministic **MQTT mock** shutdown and a **per-worker MQTT broker** for bitdeer, removing worker-test hangs.
- UI session-expiry logout/redirect, abort-on-unmount, and PDU-tab guards.

## Testing

- Backend coverage lifted above the 80% per-package gate across the client and the antminer, avalon, whatsminer, f2pool, abb, and bitdeer workers, plus new Kernel suites (including key-file integration) and devkit branch coverage.

# Public API surface (React)

Machine-readable inventory of every public export from the **React** UI
packages, generated from TypeScript source via [TypeDoc].

> **Status: experimental.** Files in this folder are committed so reviewers,
> docs validators, and historical comparisons can read them â€” but the normalizer
> that produces the `*.public-surface.json` files is not yet under test, and
> there is no CI gate enforcing regeneration. Treat the asset as advisory until
> both land.

## Scope

This folder describes the **React** public surface only â€” the contents of
`@tetherto/mdk-core-ui` and `@tetherto/mdk-foundation-ui` as exported from the
packages under `ui-client/packages/`.

Future framework targets (Vue, Svelte, etc.) are expected to live in **sibling**
top-level folders alongside `ui-client/`, each carrying its own `api/`
directory generated the same way:

```
mdk/
  core/                       framework-agnostic core (out of scope here)
  ui-client/   api/...        React (this folder)
  ui-vue/      api/...        future
  ui-svelte/   api/...        future
```

The framework is also encoded inside every JSON document via `_meta.framework`
so a consumer reading the file in isolation can identify it without knowing the
on-disk path.

## What's here

| File | Generated by | Purpose |
| --- | --- | --- |
| `core.typedoc.json` | TypeDoc directly | Full reflection dump for `@tetherto/mdk-core-ui`. |
| `core.public-surface.json` | This repo's normalizer on top of TypeDoc | One-row-per-export view of the core package. |
| `foundation.typedoc.json` | TypeDoc directly | Full reflection dump for `@tetherto/mdk-foundation-ui`. |
| `foundation.public-surface.json` | This repo's normalizer on top of TypeDoc | One-row-per-export view of the foundation package. |

The normalized `public-surface.json` files are the artefact most consumers will
read. The raw `typedoc.json` dumps are committed alongside so the normalizer
can be tuned without re-invoking TypeDoc.

## How to regenerate

```bash
# from ui-client/
pnpm api:surface
```

The generator script lives at
[`scripts/generate-public-surface.mjs`](../scripts/generate-public-surface.mjs).
It bootstraps TypeDoc against each package barrel, writes the raw dump, walks
the project reflection, and emits the normalized JSON. Re-run it whenever a
public type, prop, or export changes.

## Two layers of trust

1. **Raw `*.typedoc.json` (TypeDoc itself).** Driven by the TypeScript Compiler
   API â€” the same one `tsc` uses. Names, signatures, JSDoc, member lists,
   optionality flags and source locations all come from a real type-checker
   pass. Treat as ground truth.

2. **Normalized `*.public-surface.json` (this repo's normalizer).** Hand-written
   classifier on top of TypeDoc. Accurate today, but not yet under test.
   - **Trust** the `name`, `package`, `namespace`, `sourcePath`, `members[]`, and
     `signature` blocks â€” those are direct passthroughs from TypeDoc.
   - **Treat as advisory** the `kind` field (see [Kind enum](#kind-enum) below).
     `component` and `hook` use heuristics; edge cases can misclassify.
   - **Runtime contract** â€” every file is validated against
     [`schema.mjs`](schema.mjs) (Zod) before write; TypeScript types live in
     [`schema.ts`](schema.ts). Downstream tools should treat those files as the
     canonical shape, not the generator implementation alone.

## Known limitations

- **Mapped/derived type aliases.** Aliases built from `Pick<â€¦>`, `Omit<â€¦>`,
  intersections, conditional types, etc. don't expose a property list to
  TypeDoc, so they appear as `aliasOf: "<type string>"` with no `members[]`
  breakdown. Per-prop docs validation is not possible against those rows.
- **Variables typed as `any`.** When an upstream dependency ships a broken or
  missing `.d.ts`, the variable's type can resolve to `any`. The normalizer
  rescues components by falling back to source-path heuristics â€” but a future
  occurrence in a non-`/components/` folder would silently degrade. See the
  `react-gauge-chart` precedent for the failure mode.
- **No CI gate.** Until one is added, regeneration on every public-API change
  is on the contributor. The diff in PR review is the only safety net.
- **Schema bumps.** Breaking shape changes increment `_meta.schemaVersion`
   (currently `"1.0"`). Consumers can pin on that field.
- **Function overloads.** Only the **first** overload signature is captured on
   `function` / `hook` rows.
- **Generic type aliases.** Bodies like `PaginatedResponse<T>` appear as
   `aliasOf` without preserving generic parameters in a structured way.
- **Parameter `required` vs defaults.** TypeDoc sometimes marks parameters with
   default values as `required: true`; compare `defaultValue` when in doubt.

These JSON files can back automated checks for the same inventory the docs describe:

| Docs area | Example paths | Surface coverage |
| --- | --- | --- |

## Kind enum

| Value | Meaning for consumers |
| --- | --- |
| `component` | React component (JSX). Heuristic: React-ish type, `/components/` path, or JSX `@example`. |
| `hook` | React hook: name matches `use[Uppercase]â€¦` **and** file under `/hooks/`. Not merged into `component`. |
| `function` | Plain function export (utilities, factories). Includes non-hook functions outside `/hooks/`. |
| `value` | Variable/`const` with PascalCase name that did not match the component heuristic. |
| `const` | Variable/`const` with non-PascalCase name (typical constants map/literals). |
| `type` | Interface or type alias. Object shapes use `members[]`; unions/mapped types use `aliasOf`. |
| `class` | `class` declaration. |
| `enum` | TypeScript `enum`. |
| `namespace` | Re-exported namespace (`export * as X`). Children list `namespace: "X"`. |

## Schema

Canonical validation: [`schema.mjs`](schema.mjs) (Zod). Type-only definitions:
[`schema.ts`](schema.ts).

Each `*.public-surface.json` document has the form:

```jsonc
{
  "_meta": {
    "status": "experimental",
    "framework": "react",
    "schemaVersion": "1.0",
    "disclaimer": "Generated. See ui-client/api/README.md before consuming.",
    "generator": "scripts/generate-public-surface.mjs",
    "typedocVersion": "0.28.19"
  },
  "package": "core",
  "generatedAt": "2026-05-10T...",
  "exports": [
    {
      "name": "GaugeChart",
      "package": "core",
      "kind": "component",
      "namespace": null,
      "sourcePath": "packages/core/src/components/gauge-chart/index.tsx",
      "summary": "GaugeChart - Presentational gauge/speedometer chart"
    },
    {
      "name": "formatNumber",
      "package": "core",
      "kind": "function",
      "namespace": null,
      "sourcePath": "packages/core/src/utils/format.ts",
      "summary": null,
      "signature": {
        "parameters": [
          {
            "name": "number",
            "required": true,
            "type": "string | number | null | undefined",
            "defaultValue": null,
            "summary": null
          }
        ],
        "returnType": "string",
        "returnSummary": null
      }
    }
  ]
}
```

### Field reference (per export row)

**Always present:** `name`, `package`, `kind`, `namespace` (`null` or string),
`sourcePath` (string; empty string when TypeDoc has no source â€” e.g. some re-exports),
`summary` (`null` or string).

**Optional, by kind:**

| Field | When |
| --- | --- |
| `members` | Object-like `type` / `enum` rows; object-literal `const` / `value` (e.g. `COLOR`, `UNITS`). Each entry: `name`, `required`, `type`, `summary`. |
| `signature` | `function` and `hook` rows. `parameters[]` each have `name`, `required`, `type`, `defaultValue`, `summary`; plus `returnType`, `returnSummary` (`@returns`). |
| `aliasOf` | `type` rows with no `members` (unions, picks, etc.). |
| `type` | `const` / `value` rows where the export is not a broken-down object literal (fallback stringified type). |

Run `pnpm api:surface` validates each document with Zod before writing.

## Roadmap to remove the "experimental" tag

1. **CI regenerate-and-diff gate** â€” `pnpm api:surface` runs in CI and fails if
   `git diff --exit-code ui-client/api/` is non-empty.
2. **Smoke assertions** â€” a small test that pins well-known landmarks
   (`LineChart` is a `component`, `LineDataset.borderColor` is required, etc.)
   and asserts no public component has `type: "any"` in the raw dump.
3. **Fixture-based snapshot test** â€” a tiny throwaway package covering each
   export style, with the normalized JSON snapshotted, to protect the
   normalizer independently of the real packages.

# Public API surface (React)

Machine-readable inventory of every public export from the **React** UI
packages, generated from TypeScript source via [TypeDoc].

> **Status: experimental.** Files in this folder are committed so reviewers,
> docs validators, and historical comparisons can read them — but the normalizer
> that produces the `*.public-surface.json` files is not yet under test, and
> there is no CI gate enforcing regeneration. Treat the asset as advisory until
> both land.

## Scope

This folder describes the **React** public surface only — the contents of
`@tetherto/mdk-core-ui` and `@tetherto/mdk-foundation-ui` as exported from the
packages under `ui-client/packages/`.

Future framework targets (Vue, Svelte, etc.) are expected to live in **sibling**
top-level folders alongside `ui-client/`, each carrying its own `api/`
directory generated the same way:

```
mdk/
  core/                       framework-agnostic core (out of scope here)
  ui-client/   api/...        React (this folder)
  ui-vue/      api/...        future
  ui-svelte/   api/...        future
```

The framework is also encoded inside every JSON document via `_meta.framework`
so a consumer reading the file in isolation can identify it without knowing the
on-disk path.

## What's here

| File | Generated by | Purpose |
| --- | --- | --- |
| `core.typedoc.json` | TypeDoc directly | Full reflection dump for `@tetherto/mdk-core-ui`. |
| `core.public-surface.json` | This repo's normalizer on top of TypeDoc | One-row-per-export view of the core package. |
| `foundation.typedoc.json` | TypeDoc directly | Full reflection dump for `@tetherto/mdk-foundation-ui`. |
| `foundation.public-surface.json` | This repo's normalizer on top of TypeDoc | One-row-per-export view of the foundation package. |

The normalized `public-surface.json` files are the artefact most consumers will
read. The raw `typedoc.json` dumps are committed alongside so the normalizer
can be tuned without re-invoking TypeDoc.

## How to regenerate

```bash
# from ui-client/
pnpm api:surface
```

The generator script lives at
[`scripts/generate-public-surface.mjs`](../scripts/generate-public-surface.mjs).
It bootstraps TypeDoc against each package barrel, writes the raw dump, walks
the project reflection, and emits the normalized JSON. Re-run it whenever a
public type, prop, or export changes.

## Two layers of trust

1. **Raw `*.typedoc.json` (TypeDoc itself).** Driven by the TypeScript Compiler
   API — the same one `tsc` uses. Names, signatures, JSDoc, member lists,
   optionality flags and source locations all come from a real type-checker
   pass. Treat as ground truth.

2. **Normalized `*.public-surface.json` (this repo's normalizer).** Hand-written
   classifier on top of TypeDoc. Accurate today, but not yet under test.
   - **Trust** the `name`, `package`, `namespace`, `sourcePath`, and `members[]`
     blocks — those are direct passthroughs from TypeDoc.
   - **Treat as advisory** the `kind` field (`component` / `value` / `type` /
     `function` / `const` / `class` / `namespace` / `enum`). It uses heuristics
     (React-component-shaped types, source path under `/components/`, JSX
     `@example` blocks) that can misclassify edge cases.

## Known limitations

- **Mapped/derived type aliases.** Aliases built from `Pick<…>`, `Omit<…>`,
  intersections, conditional types, etc. don't expose a property list to
  TypeDoc, so they appear as `aliasOf: "<type string>"` with no `members[]`
  breakdown. Per-prop docs validation is not possible against those rows.
- **Variables typed as `any`.** When an upstream dependency ships a broken or
  missing `.d.ts`, the variable's type can resolve to `any`. The normalizer
  rescues components by falling back to source-path heuristics — but a future
  occurrence in a non-`/components/` folder would silently degrade. See the
  `react-gauge-chart` precedent for the failure mode.
- **No CI gate.** Until one is added, regeneration on every public-API change
  is on the contributor. The diff in PR review is the only safety net.
- **Schema may change.** The shape of `*.public-surface.json` is not yet a
  stable contract. Expect tweaks as the normalizer is hardened.

## Schema

Each `*.public-surface.json` document has the form:

```jsonc
{
  "_meta": {
    "status": "experimental",
    "framework": "react",
    "disclaimer": "Generated. See ui-client/api/README.md before consuming.",
    "generator": "scripts/generate-public-surface.mjs",
    "typedocVersion": "0.28.19"
  },
  "package": "core",
  "generatedAt": "2026-05-10T...",
  "exports": [
    {
      "name": "GaugeChart",
      "package": "core",
      "kind": "component",
      "namespace": null,
      "sourcePath": "packages/core/src/components/gauge-chart/index.tsx",
      "summary": "GaugeChart - Presentational gauge/speedometer chart"
    },
    {
      "name": "GaugeChartProps",
      "package": "core",
      "kind": "type",
      "namespace": null,
      "sourcePath": "packages/core/src/components/gauge-chart/index.tsx",
      "summary": null,
      "members": [
        {
          "name": "percent",
          "required": true,
          "type": "number",
          "summary": "Value between 0 and 1 (e.g. 0.75 = 75%)"
        }
        // …
      ]
    }
  ]
}
```

Field reference (per row):

| Field | Always present | Notes |
| --- | --- | --- |
| `name` | yes | Exported name. |
| `package` | yes | `"core"` or `"foundation"`. |
| `kind` | yes | Coarse classification (advisory — see "Two layers of trust"). |
| `namespace` | yes | `null` at top level, namespace name when nested (e.g. `"DropdownMenu"`). |
| `sourcePath` | yes | Path relative to `ui-client/`. |
| `summary` | yes | First line of the JSDoc summary, or `null`. |
| `members` | type/enum rows with object-literal shape | Array of `{ name, required, type, summary }` for each property. |
| `aliasOf` | type rows without a member breakdown | TypeDoc's stringified type expression. |
| `type` | const / value rows | TypeDoc's stringified resolved type. |

## Roadmap to remove the "experimental" tag

1. **CI regenerate-and-diff gate** — `pnpm api:surface` runs in CI and fails if
   `git diff --exit-code ui-client/api/` is non-empty.
2. **Smoke assertions** — a small test that pins well-known landmarks
   (`LineChart` is a `component`, `LineDataset.borderColor` is required, etc.)
   and asserts no public component has `type: "any"` in the raw dump.
3. **Fixture-based snapshot test** — a tiny throwaway package covering each
   export style, with the normalized JSON snapshotted, to protect the
   normalizer independently of the real packages.
